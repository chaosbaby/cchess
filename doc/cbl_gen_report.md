  1. 成功步骤总结

   1. 实物样本比对 (Binary Diff-Driven)：
      通过对比用户提供的 misk/ 目录下“由象棋桥生成的真实文件”与“代码生成的文件”，发现了关键的偏移量位移。
   2. 阶梯偏移算法校准 (Staggered Offsets)：
      确认了象棋桥库文件容量超过 128 局时，头部大小会从 101,952 字节发生阶梯状跃迁。
   3. 摘要表 (Summary Table) 精度校准：
      这是最核心的步骤。通过对摘要表每一项（276 字节）的逐位排查，精确定位了索引、激活位、UUID 和标题的物理偏移。
   4. 去噪验证法 (Noise-Free Verification)：
      通过生成仅包含“一步棋（炮二平五）”的极简
  verify_minimal_1.cbl，排除了因招法编码复杂度导致的干扰，确保了底层文件格式逻辑的绝对正确。

  2. 核心技术要点 (Success Keys)

  ┌──────────────────┬─────────────────┬─────────────────────────────────────────────────────────────────┐
  │ 关键项           │ 偏移量 (Offset) │ 成功规格说明                                                    │
  ├──────────────────┼─────────────────┼─────────────────────────────────────────────────────────────────┤
  │ Header Version   │ 16              │ 必须写入 03 00 00 00。                                          │
  │ Header Flag      │ 52              │ 必须写入 FF FF FF 7F 以激活高级特性。                           │

  │ Summary UsedSize │ Summ + 12       │ 关键点：写入该局棋谱在 CBR 块中的实际有效字节长度（含着法区）。 │
